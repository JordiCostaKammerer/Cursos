Detalles de la tarea de esta unidad.
Enunciado.
Per tal de treballar des del teu ordinador (client) amb una base de dades remota (servidor), crea un aplicació web en Java a la teva màquina amfitriona (client) amb un IDE (Eclipse, Netbeans, VSC o IntelIJ) i instal·la una base de dades MariaDB (codi adjunt) a la màquina virtual remota (servidor Debian). El projecte ha d'incloure:

A la part client:

Llibreria necesària a l'IDE per programar amb Servlets.
Llibreria JDBC a l'IDE que permeti connectar la vostra aplicació a una base de dades remota.
Un arxiu JSP "botiga.jsp" a l'IDE que permeti mostrar la consulta a la base de dades "la_meva_botiga" (codi adjunt). L'aplicació ha de mostrar la taula dels productes en html i bootstrap i carregar aquest arxiu per defecte quan es desplegui. Les llibreries Bootstrap han de formar part com a carpetes al projecte.
Heu de fer la inserció de, almenys, un producte a la base de dades. En aquest sentit, l'arxiu JSP ha de carregar el servlet "Formulari" i aquest ha de cridar al servlet "Insercio", per tal que redireccióni a botiga.jsp i mostri la taula amb el producte nou introduït. Els servlets s'han de connectar a la base de dades remota mitjançant la classe de java "Connexio", on hi haurà les credencials per fer la connexió al servidor remot.
Cal incorporar el servidor Tomcat a l'IDE de la màquina amfitriona (client), per tal de fer els desplegaments i fer-ne les republicacions necessàries i veure els canvis que es van produint.
Cal incorporar a l'IDE la connexió a la base de dades remota (servidor). Cal fer "ping" des de l'IDE per comprovar que se n'ha establert la connexió.
Instal·lar a l'amfitrió el gestor de base de dades Worckbench i fer la connexió remota a la base de dades del servidor per tal de comprovar la inserció o insercions fetes a la base de dades.
Una vegada acabada l'aplicació a la màquina amfitriona (client), cal exportar-la a un arxiu .war (Botiga.war).

A la part servidor:

Finalment, cal instal·lar a la màquina remota un servidor WildFly, crear un usuari administrador i desplegar l'arxiu Botiga.war.
NOTA: s'ha d'entregar un document PDF amb les captures necessàries que expliquin tot el procés ben documentat. Les captures s'han d'extreure d'un vídeo explicatiu (màxim de 10 minuts) que expliqui cada passa (llibreries, arxiu JSP i servlets, servidor Tomcat, base de dades a l'IDE, Workbend i desplegament a Wildfly). Tot plegat comprimt tot a un arxiu .zip o enllaçat el vídeo amb una URL al PDF.
